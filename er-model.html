Reflections

Before I even started coding this project, I started off doing UI prototype sketches on pieces of paper to get an idea of the different design choices I had (in terms of the UI). After this project I realized the importance of paper prototyping and its impact on making an acceptable interface design.

I've always generally known what the mode-view-controller design pattern was about and the concept of how it works, but I always found it to be confusing when I looked at code that used it. This all changed when I implemented this app with MVC, everything started piecing together and eventually it all made sense.

Learning how layout managers (BoxLayout, Strut and Glue, ect) work and how they differ from each other was also a really interesting experience. The way you use these layout managers have a big impact on how your UI looks and feels so it was nice experiencing and apply these concepts in a practical way! I also learned about affine transformation matrices and used them in this app to implement zooming in/out.

Overall this was a really enjoyable project and I learned about a lot of concepts and how to apply them.
